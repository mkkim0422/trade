# 강남구 일반음식점 폐업 위험 분석 시스템 - 거버넌스

## §1. 프로젝트 개요

### 1-1. 목적
강남구 기반 일반음식점 폐업 위험도 분석 GIS 시스템.
타겟: 대형 프랜차이즈 점포개발팀장 (B2B).
차별화: "여기 망할 수도 있다"를 데이터로 진단하는 유일한 도구.

### 1-2. 기술 스택
- Next.js 15 (App Router), TypeScript, Tailwind CSS
- Zustand (전역 상태 관리)
- 카카오 맵 SDK + Geocoding REST API
- 클러스터링: MarkerClusterer (성능 최적화)

### 1-3. 데이터
- 원본: LOCALDATA 2024.10 (강남구 일반음식점, 행정구역 코드 11680)
- 가공: 카카오 Geocoding API로 좌표 변환
- Mock: 폐업일자 2025-01-01 ~ 2026-03-31 랜덤 생성 (MVP용)

## §2. 파일 크기 원칙
- 단일 파일 최대 300라인 (컴포넌트/라이브러리)
- 초과 시 분할 (Card 컴포넌트, 분석 로직 등)

## §3. 용어 표준

### 3-1. UI vs 코드
- UI 표시: 한글 (폐업 위험도, 장수 매장 분포)
- 내부 코드: 영문 (LUD, SEG, Trend, Narrative)

### 3-2. 업종 매핑
- 일반음식점 ↔ LOCALDATA 업태명
  - 한식, 중국식, 일반, 경양식, 일식 등
- (향후) 편의점 ↔ LOCALDATA 업종코드: G47122

## §4. 색상 규약
- 위험도: 적색 (#EF4444 - red-500)
- 안전/장수: 녹색 (#10B981 - green-500)
- 중립: 슬레이트 (#64748B - slate-600)
- 클러스터: 반투명 적색 (rgba(239, 68, 68, 0.8))

## §5. 데이터 정직성 원칙
- 모든 추정/Mock 데이터에 배지 표시 필수
- Mock 배지: 황색 배경 (#FEF3C7 - amber-50)
- 푸터에 데이터 기준일 명시: "LOCALDATA 2024.10 | Mock 폐업일자 (MVP)"

## §6. 레이아웃 원칙
- 3단 구조: 좌패널(370px) + 지도(flex-1) + 우패널(320px)
- flex-col / grid로 격리, 절대 float 사용 금지
- z-index 명시적 관리: 로딩(10), 카운트(10), 필터(20)

## §7. 성능 최적화
- 대량 마커: MarkerClusterer 적용 (minLevel: 5)
- useMemo로 분석 결과 캐싱 (LUD/SEG/Trend)
- Zustand로 불필요한 리렌더링 방지
